# ko

Ko's personal opinionated CLI. Thin wrappers around SDKs I use often, built so both I and AI agents (Claude Code via bash) can lean on it.

## Subcommands
- `ko exa search|get` тАФ Exa semantic web search + URL тЖТ markdown
- `ko arxiv search|fetch` тАФ arxiv search (**relevance-ranked**; `--recent` = newest-first for browsing a `cat:cs.LG`) + paper-to-markdown. For cross-publisher relevance, prefer `ko papers search`.
- `ko gsheets info|tabs|get|find | set|put|header|add-tab|new|clear|auth` тАФ read **& write** Google Sheets via OAuth. Accepts a URL or bare ID. Reads use the readonly scope; writes need read+write (`ko gsheets auth` grants it by default). Shape-aware writers (`set`/`put`, library `write_values`/`write_ranges`/`write_df`) refuse to clobber non-empty cells unless `--overwrite` тАФ the error lists them. `write_df` duck-types polars **or** pandas (neither is a dep). Generic port from the NibbleEdge `ne` tool; no project-specific shortcuts.
- `ko doc <file>` тАФ PDF/Office/image тЖТ plain text via liteparse (local, no models). Bare shortcut: `ko <file>` routes here when the arg is an existing file.
- `ko fetch <url>` тАФ URL тЖТ markdown, deterministic routing: arxivтЖТarxiv2md, YouTubeтЖТtranscript (via `ko yt`; fails loud when captions are off тАФ no silent fall-through to the useless description page), PDFтЖТ~/Downloads+liteparse (`--no-save`), doi.orgтЖТOA copy, else trafilatura, dead/emptyтЖТWayback (`--archive` forces). Bare shortcut: `ko <url>`.
- `ko yt <url|id>` тАФ YouTube тЖТ transcript via **youtube-transcript-api** (free, keyless, no video download; prefers manual subs over auto-captions). Plain text with paragraph breaks (>2s gaps); `--json` = `[{start, text, duration}]` snippets; `-s/--summarize` = one-shot LLM summary (`-m` overrides model). Accepts watch/youtu.be/shorts/live/embed URLs or a bare 11-char id. Stderr notes the ~token count. Future fallback for caption-less videos: Gemini native video understanding (URL as `file_data`, fractions of a cent) тАФ package-first because it's free.
- `ko llm "<prompt>"` тАФ one-shot LLM, stdin-aware, never has tools. Default `google:gemini-3.5-flash` (`KO_DEFAULT_MODEL`/`-m`; `-m` tab-completes models whose env key is set). OpenRouter's catalog is fetched from its public `/models` and cached at `~/.cache/ko/openrouter_models.json` (24h TTL) so `-m openrouter:<slug>` completes.
- `ko models [--refresh]` тАФ list model strings usable with `-m`, one per line. `--refresh` force-refetches the OpenRouter catalog past the 24h cache (for grabbing a just-released model).
- `ko brief [--raw] [-m model]` тАФ morning brief: deterministic gather (today's calendar + unread gmail + `hn top` + `hf top`, each best-effort тАФ an unconfigured source becomes a note, never a crash) тЖТ **one** cheap-model synthesis (scannable markdown: what's on, mail that matters, the 2-3 items worth a click). `--raw` prints the gathered sections and skips the LLM entirely. A scripted pipeline, NOT an agent loop. тЪая╕П deliberate: sends calendar/email content to the configured LLM (noted in help).
- `ko logs [--n] [--errors] [--json]` тАФ recent local command logs. ko writes **one structured JSON event per command** via **loguru** (`~/.local/state/ko/logs/ko.jsonl`, serialized JSONL, weekly rotation: cmd label, duration, exit code, error type тАФ **no args/secrets**, following PostHog's "one wide event per request"). loguru's default stderr sink is removed (ko's own output stays clean), and its sink model makes adding a sync trivial тАФ a PostHog sync is just another `logger.add(sink)`. On by default; `[logs] enabled = false` (or `[logs] level = "debug"`) in config.toml. Best-effort тАФ never breaks a command.
- `ko billing` тАФ read-only balance + recent usage across paid services. v1: OpenRouter credits ($ remaining of total purchased; today/week/month usage). `--json`. One function per provider in `billing.py` (degrades gracefully like `ko doctor` тАФ a missing key is a note, not a crash). pydantic-ai has no account-billing API, so we hit each provider's REST API directly.
- `ko hn top|search|item` тАФ Hacker News via Algolia (no auth). `top` = hckrnews-style top 10/20 by points (`--now` = the live front page via Algolia's `front_page` tag тАФ complementary: the tag only marks the *current* page, `top` does past windows); `search` defaults to last 12 months; `item` = story + comment tree as text.
- `ko hf top|search|info|get` тАФ Hugging Face paper pages (no auth). `top` = Daily Papers in HF's *trending* order (upvote-driven with recency decay тАФ not strictly upvote-sorted); `search` = semantic; `info` = metadata incl. github/models/datasets; `get` = paper as markdown. Same ids as arxiv тАФ composes with `ko arxiv fetch`.
- `ko papers search|get|cites|refs|similar` тАФ cross-publisher literature scouting via **OpenAlex** (no key; `mailto` polite pool; its `oa_url` IS Unpaywall's data). Takes DOIs, arxiv ids, or OpenAlex W-ids interchangeably. `search` = all publishers (TSV: year, cites, oa_status, title, doi, journal); `get` = full text via the OA copy + fetch pipeline, else a metadata card (authors, journal, S2 tldr, abstract reconstructed from `abstract_inverted_index`) тАФ paywalled тЙа error; `cites`/`refs` = citation graph, most-cited first; `similar` = S2 recommendations (needs free `S2_API_KEY` тАФ keyless S2 429s immediately, so tldr/similar degrade gracefully without it). Plus DOI routing in `ko fetch`: doi.org URLs resolve the OA copy first instead of 403ing on publisher landing pages. Design: `docs/papers-cli-design.md`. Known wart: a few OpenAlex DOI records are bad merges (wrong title, right abstract) тАФ `search` for the title and use the W-id when a record looks off.
- `ko tv <title>` тАФ movie/TV rating + overview + regional watch providers (`TMDB_READ_ACCESS_TOKEN`, free; AU default, `--country`).
- `ko x search|list|user|lists` тАФ X via the official XDK (`X_BEARER_TOKEN`). `list <name|id|url>` = recent posts from a list (name = one of mine; a bare list id or `x.com/i/lists/<id>` URL reads **any public list**); bare shortcut `ko x ai` тЙб `ko x list ai`. `user <@handle|url>` = one user's timeline (further back than search's ~7-day window). `lists` = my lists, biggest first, with descriptions + member counts (owned lists; *followed* lists need a higher tier). `search` тЙд7 days uses the recent index; `--days >7` switches to **full-archive** (years back); `--list <name|id|url>` scopes a search to a list (`ko x search "claude code" --list ai --days 90`). **Likes/bookmarks/home-timeline aren't available** тАФ they need OAuth user-context, not the app-only bearer (which also can't self-identify, so set your handle via `[x] handle` in config.toml or `KO_X_HANDLE`; default `ko`). **Cost**: X is pay-per-use since 2026-02 (prepaid credits) тАФ ~$0.005/post read (owned reads like `lists` $0.001), 2M reads/mo cap; so `ko x list --n 20` тЙИ $0.10, `ko x lists` тЙИ a cent. Full-archive/big `--n` cost more тАФ mind them. [pricing](https://docs.x.com/x-api/getting-started/pricing)
- `ko tt lists|items <list>` тАФ TickTick (read-only) via its hosted MCP (`TICKTICK_API_KEY`, env or config.toml). `lists` = your lists; `items <list>` = open tasks (name substring or id). Bare shortcut `ko tt <list>` тЙб `ko tt items <list>`. Read-only by design тАФ manage tasks in TickTick itself. We're an MCP *client* here (`mcp` SDK over Streamable HTTP).
- `ko gdocs get|push|replace|append|comments|reply|shade-table` (alias `ko gd`) тАФ Markdown тЖФ Google Docs via OAuth (shared token; Docs + Drive `drive.file` scopes). `get` = DocтЖТMarkdown (Drive export, tables included, pipe it); `push <file> [doc]` = MarkdownтЖТnew Doc, or update an existing one in place (shows a diff + confirm, `--force` skips); `replace`/`append` = in-place edits; `comments`/`reply` = Drive comments; `shade-table` = header/totals cell shading (the bit Markdown can't do). What converts: see `docs/gdocs-markdown.md`.
- `ko cal [agenda] | day | find | add | cals` тАФ Google Calendar agenda + quick-add (shared token; `calendar.readonly` + `calendar.events`). Bare `ko cal` = next 7 days across calendars (`--days`/`--today`); `find <text>` = events matching a title substring, forward or `--past` ('when was my last X'; borrowed from chota-bot); `add "title" <when>` accepts ISO / today / tomorrow (no date lib). Times in `[cal] timezone` (default Australia/Sydney). Minimal тАФ no edit/delete/RSVP.
- `ko gmail [recent] | search | from | view | thread` тАФ read Gmail (**read-only**; shared token, `gmail.readonly`). Bare `ko gmail` = recent inbox, one line/msg (id, date, from, subject + snippet; `-n`, `--unread`); `search "<query>"` passes Gmail's own syntax verbatim (`from:`/`newer_than:7d`/`is:unread`); `from <who>` shortcut; `view <id> [--full]` = one message; `thread <id> [--full]` = the whole conversation, oldest first. `--json` everywhere. Concise on purpose (vs the verbose Gmail MCP). No send/labels/drafts.
- `ko publish new <dir> [--md|--bare|--hono]` / `ko publish preview [dir]` / `ko publish [dir]` / `ko publish list` тАФ publish a folder to Cloudflare. `preview` runs `wrangler dev` locally (real http; `file://` breaks ES modules + fetch). `new` scaffolds: default static (Tailwind+Alpine), `--md` (markdown doc site тАФ generic `?page=` shell, README hub, sidebar TOC), `--bare` (just a CLAUDE.md of hints). Bare `ko publish [dir]` deploys; the folder's own `wrangler.jsonc` makes the name sticky тЖТ re-publish overwrites the same URL. Custom domain defaults to `<name>.khalido.dev` (`DEFAULT_DOMAIN`; override via `[publish] domain`). Creds: `KO_CLOUDFLARE_API_TOKEN`/`KO_CLOUDFLARE_ACCOUNT_ID` тЖТ config.toml тЖТ standard `CLOUDFLARE_*`. **Won't silently take over** an existing Worker name (checks the account; `--force` to override). Needs **wrangler** (pinned repo-local via `npm install`; `KO_WRANGLER` тЖТ `node_modules/.bin` тЖТ PATH тЖТ `npx wrangler@4`). `--hono` adds a Hono worker (API routes, D1/R2, optional `--pin` gate, cached `/api/data`); `ko publish --pin new` rotates the gate PIN. Cloudflare Access private pages + AI on sites deferred тАФ see `docs/publish.md` / `docs/publish-ai.md`.
- `ko agent research|tv|sessions` тАФ pydantic-ai agents. `research` (web+papers+HN, capable model), `tv` (movies/TV in AU, cheap model), no prompt = interactive REPL, `-m` overrides model, `-r <id>` resumes a saved session. `sessions` lists saved sessions (title/summary/tags once summarized; `--tag`/`--search` filter); `sessions summarize [--n] [-m]` = cheap-model title + one-line takeaway + tags per session into the SQLite index. **`ko a тАж` is a hidden shorthand for `ko agent тАж`** (e.g. `ko a research "тАж"`). See Agents below.
- `ko mcp inspect <name|url>` / `ko mcp call <name|url> <tool>` / `ko mcp overview <name|url>` / `ko mcp servers` тАФ investigate/use MCP servers (ko is itself an MCP client). `overview` runs a pydantic-ai agent that connects (via `load_mcp_toolsets` тАФ same mcp.json shape) and summarizes what the server is for + what's useful (reads only, won't call mutating tools; cheap model, `-m` to override) тАФ handy for sizing up your own dev server. Target a **server name** from `~/.config/ko/mcp.json` (the standard `{"mcpServers": {...}}` shape тАФ copy-paste-compatible with Claude Code/opencode; supports both `url`+`headers` and stdio `command`/`args`; secrets use `${ENV_VAR}` placeholders expanded from env/config at load, so the file is safe to check in) **or a raw URL**. `inspect` lists tools + resources + prompts (+ capabilities + the server's instructions); `--tool <name>` dumps that tool's full JSON input schema. `call` invokes a tool (`--arg k=v`). `servers` lists the registry. **`auth-info <url>`** is spec-aware (RFC 9728 + 8414): it walks a remote server's OAuth discovery surface *before* connecting тАФ protected-resource + auth-server metadata, scopes, endpoints, DCR тАФ and flags anything malformed (e.g. HTML where JSON should be); exit 1 on problems. `-H` overrides/adds headers, `--json` everywhere. **On failure** it prints the server's real HTTP status + body, the `WWW-Authenticate` header (the server saying *why*), and тАФ if a Bearer token was passed тАФ the token's decoded `aud/iss/scope/exp` claims (a wrong audience is otherwise invisible). (A 503 "not configured" = server-side, not your client.) A verb takes a name/url = client; a future `ko mcp serve` (no target) = ko as server. Listings to stdout, banner to stderr.
- `ko prompt [name]` тАФ kickoff briefs: my opinionated "how I build X" notes, pulled by name to load into an agent ("get `ko prompt sveltekit-app`") or copy-paste (`--copy`). Bare `ko prompt` lists (TSV: name, description); `ko prompt <name>` prints the markdown. Briefs ship in `src/ko/prompts/*.md`; add or override per machine in `~/.config/ko/prompts/*.md` (a user file wins by name). A meta-layer on top of generic llms.txt тАФ my picks + the gotchas, with links out to the canonical docs. The tool is just a thin loader; the content is hand-written.

## Principles
- **Opinionated, not generic.** Do what I actually do, the way I do it. Defaults match my habits.
- **Thin wrappers.** Each module (`exa.py`, `arxiv.py`, `gsheets.py`) is a self-contained domain module over one upstream SDK. Keep parameter names matching the SDK so agent-written code composes with upstream docs.
- **Help lines carry their weight.** Every subcommand and flag has a concise, useful `help=`. Assume humans and agents skim `--help` and nothing else.
- **Human-readable and pipeable by default.** Plain text / TSV. Add `--json` when structure matters. Errors to stderr. `typer.Exit(0)` for "no results" (not an error).
- **One file per domain.** `cli.py` registers subapps; each module owns its logic. No shared `utils/` until three things need the same thing.
- **`--help` is the contract.** Machine-friendly output is a design constraint, not an afterthought.

## Agents (`src/ko/agents/`)
- **Tools declared once** as `FunctionToolset`s in `_toolsets.py` (`web` = exa+fetch, `papers` = arxiv+hf+OpenAlex citation-graph [search/get/cites/refs], `news` = hn [top/front_page/search/discussion тАФ search passes through since_months/by_date/min_comments], `tmdb`); agents compose the subset they need. **Docstring-as-contract**: a tool docstring must state any silent filtering default of the underlying function (date windows, result caps, sort orders, index coverage) тАФ the agent-side twin of "--help is the contract". The research agent's instructions encode a snowball workflow (seed тЖТ cites/refs тЖТ saturation); see `ko prompt research-papers` for the human-facing version. A toolset is stateless and shared by reference; each carries its own `instructions=`.
- **One file per agent** (`research.py`, `tv.py`): ~15 lines тАФ `Agent(model, instructions=..., toolsets=[...])`. Per-agent default model (research = capable, tv = cheap); `-m`/`KO_AGENT_MODEL` overrides per run (passed to the run call, not baked into the Agent тАФ the Agent reads its default at import).
- **`_shared.py`** тАФ generic run/stream/repl over any Agent; streams pretty markdown to a TTY, plain text when piped.
- **Resilient tools** тАФ each wraps its call in `_try`, returning an error note instead of raising, so one flaky source (rate limit, timeout) doesn't abort a run; the model routes to another source.
- **Sessions** (`src/ko/sessions.py`) тАФ every turn dumps `all_messages()` (full tool trace) to `~/.local/state/ko/sessions/<id>.json` via pydantic-ai's `ModelMessagesTypeAdapter`. Resume with `-r <id>`; list with `ko agent sessions`. Flat files in state (not config тАФ generated, not hand-edited).
- **Session index** (`~/.local/state/ko/ko.db`, SQLite) тАФ `ko agent sessions summarize` writes one row per session (title тЙд8 words, one-sentence takeaway, 3-5 tags) via a cheap-model structured-output call; `sessions --tag/--search` filter on it. The JSON files stay the source of truth тАФ the DB is a rebuildable projection ("the log is the agent"). The model sees a **deterministic digest**, not the raw log: Q&A verbatim, each tool call compressed to one `[tool: name(argsтАж) тЖТ returnтАж]` line; trimming drops tool lines first, middle turns next, never the first prompt or final answer.
- **No YAML agent-spec**: pydantic-ai specs can't reference custom toolsets, so code factories win. Capabilities (lifecycle hooks) are deferred until we need request/tool interception.

## Adding an agent
1. Add/extend a `FunctionToolset` in `agents/_toolsets.py` if new tools are needed.
2. `agents/<name>.py` тАФ `Agent(model, instructions=..., toolsets=[...])` + thin `run`/`repl` binding `_shared`.
3. Export in `agents/__init__.py`; add a `@agent_app.command` in `cli.py`.

## Google auth (gsheets)
- OAuth user flow (desktop app). First command triggers browser consent, token cached at `~/.local/state/ko/google_token.json`.
- `ko gsheets auth` (or `gdocs`/`cal`/`gmail auth` тАФ same flow) grants **one read+write token per account** covering Sheets + Docs + Calendar (write) and Gmail (**read-only**). **No Drive scope** тАФ can't browse Drive, only docs/sheets by ID. Reads use the narrower readonly scopes; the one token serves all. `--readonly` narrows; upgrade an existing token with `auth --logout` then `auth`. Adding another API later = one re-consent.
- Personal Gmail: an **External** app left in *Testing* expires the refresh token after **7 days** тАФ set the consent screen to **Internal** (Workspace) or **Publish** the app. The token file is portable across machines (copy it, or share via `KO_STATE_DIR`).
- **Multi-account.** Active account = `-a/--account` (any `ko gsheets` cmd, sets `KO_GOOGLE_ACCOUNT`) тЖТ `[google] account` тЖТ `"default"`. Per-account token `google_token_<account>.json` (legacy `google_token.json` for `default`); per-account client `google_client_<account>.json` falls back to the shared `google_client.json` (only needed when one OAuth client can't authorize an account, e.g. a Workspace-Internal consent screen + a personal Gmail). `ko gsheets accounts` lists authed accounts. All paths via `google_auth.token_file()/client_file()/active_account()`.
- **Scope is per-API, not per-folder.** OAuth grants the tool *your* access within the scope (now Sheets only). It can't be limited to specific folders/files; true per-file scoping needs a service account (share files with its email) тАФ a separate model not used here.
- Prereq: OAuth 2.0 Desktop client JSON at `~/.config/ko/google_client.json` (or set `KO_GOOGLE_CLIENT_FILE`). Full setup in README.

## Directories (`src/ko/dirs.py`)
- `~/.config/ko/` тАФ user-editable config, dotfile-sync-safe (`KO_CONFIG_DIR`). `config.toml` `[keys]` table supplies API keys/tokens as a fallback to env vars (env always wins); loaded into the environment at startup so SDKs that read `os.environ` directly pick them up. `ko doctor` shows each key's source: env / config / missing.
- `~/.local/state/ko/` тАФ tokens + id caches, never synced (`KO_STATE_DIR`); old config-dir state files auto-migrate
- `~/.cache/ko/` тАФ disposable (`KO_CACHE_DIR`)
- `ko doctor` shows per-tool setup status (keys set, binaries found, auth done)

## Agent notes
- **`AGENTS.md`** at the repo root is the agent-facing contract (output/exit-code/`--json` conventions, bare-arg shortcuts). Keep it in sync with the bullets below.
- `ko --help` / `ko <cmd> --help` are the contract
- `ko gsheets get ... ` defaults to TSV тАФ `cut -f N` safe
- `--json` on most read commands тЖТ a JSON array of objects (a few return a single object; `gsheets get` is a 2D array). Under `--json`: errors are `{error, code}` on stderr, empty results are `[]` on stdout. stdout = data, stderr = notes/errors.
- Exit codes: `0` success (incl. empty results), `1` runtime error, `2` usage error
- No interactive prompts inside commands (except the one-time OAuth browser popup)
